# Story: E5-S5 - Connect Recording Flow to Note Saving

## Story Info
- **Epic:** E5 - Note Management
- **Sprint:** 3
- **Priority:** Must Have
- **Points:** 3
- **Status:** not_started
- **Dependencies:** E5-S1

## Description

As a user, I need the complete flow from recording to saved note to work end-to-end so that my voice notes are captured, processed, and saved seamlessly.

## Acceptance Criteria

- [ ] Full flow works: Record -> Stop -> Process -> Preview -> Save -> Home
- [ ] Save button on Preview screen calls `createNote` with:
  - `title`: from preview (possibly edited)
  - `formatted_text`: from preview (possibly edited)
  - `raw_transcription`: from processing result
  - `format_type`: from user's default preference
- [ ] After save, navigate to Home screen
- [ ] New note appears at the top of the grid immediately
- [ ] Format type from user's preference (PreferencesContext) is passed to processing
- [ ] Success feedback shown (brief toast or visual confirmation)
- [ ] Error on save shows error message with option to retry

## Technical Notes

### Wiring the Flow

**Record screen** (`app/record.tsx`):
```typescript
const { defaultFormat } = usePreferences();

const handleStop = async () => {
  const uri = await stopRecording(recording);
  router.push({
    pathname: '/preview',
    params: { audioUri: uri, formatType: defaultFormat }
  });
};
```

**Preview screen** (`app/preview.tsx`):
```typescript
const { createNote } = useNotes();

const handleSave = async () => {
  try {
    await createNote({
      title: editableTitle,
      formatted_text: editableText,
      raw_transcription: result.transcription,
      format_type: formatType,
    });
    router.replace('/');
  } catch (error) {
    setError('Failed to save note. Please try again.');
  }
};
```

### Navigation
Use `router.replace('/')` after save so the user cannot navigate back to the preview screen.
