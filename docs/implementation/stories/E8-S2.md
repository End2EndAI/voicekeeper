# Story: E8-S2 - Link Audio URL to Notes

## Story Info
- **Epic:** E8 - Audio Storage
- **Sprint:** 4
- **Priority:** Should Have
- **Points:** 2
- **Status:** not_started
- **Dependencies:** E8-S1

## Description

As a user, I want my notes to be linked to their original audio so that I can tell which notes have recordings available.

## Acceptance Criteria

- [ ] After successful audio upload, note record updated with `audio_url`
- [ ] Upload happens in the background after note save (non-blocking)
- [ ] NoteCard shows audio indicator icon when `audio_url` exists
- [ ] Note Detail screen shows audio indicator when `audio_url` exists
- [ ] Notes saved even if audio upload fails (audio_url remains null)

## Technical Notes

### Background Upload Flow
```typescript
// In preview.tsx handleSave
const handleSave = async () => {
  // 1. Save note first (fast, critical path)
  const note = await createNote({ title, formatted_text, raw_transcription, format_type });

  // 2. Navigate home immediately
  router.replace('/');

  // 3. Upload audio in background (non-blocking)
  if (audioUri) {
    const audioUrl = await uploadAudio(audioUri, user.id, note.id);
    if (audioUrl) {
      await supabase.from('notes').update({ audio_url: audioUrl }).eq('id', note.id);
    }
  }
};
```

### Audio Indicator
```typescript
// In NoteCard
{note.audio_url && (
  <View style={styles.audioIndicator}>
    <MicIcon size={12} />
  </View>
)}
```
