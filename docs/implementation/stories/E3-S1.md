# Story: E3-S1 - Microphone Permission Handling

## Story Info
- **Epic:** E3 - Voice Recording
- **Sprint:** 2
- **Priority:** Must Have
- **Points:** 2
- **Status:** not_started
- **Dependencies:** None

## Description

As a user, I need the app to properly request and handle microphone permissions so that I can record voice notes reliably.

## Acceptance Criteria

- [ ] Microphone permission requested on first recording attempt (not on app launch)
- [ ] If permission granted, proceed directly to recording
- [ ] If permission denied, show explanation screen with:
  - Clear message explaining why microphone access is needed
  - Button to open system settings (Linking.openSettings)
  - Button to go back to Home
- [ ] Permission status checked before every recording attempt
- [ ] Works correctly on iOS, Android, and web

## Technical Notes

### Permission Request
```typescript
import { Audio } from 'expo-av';

const requestPermission = async (): Promise<boolean> => {
  const { status } = await Audio.requestPermissionsAsync();
  return status === 'granted';
};

const checkPermission = async (): Promise<boolean> => {
  const { status } = await Audio.getPermissionsAsync();
  return status === 'granted';
};
```

### Web Considerations
On web, the browser handles permission prompts natively. The `expo-av` API abstracts this, but the UX may differ slightly.

## Files to Create/Modify
- `services/recording.ts` - Permission request and check functions
- `app/record.tsx` - Check permission on mount, show denied state if needed

## Testing
- First recording attempt triggers permission dialog
- Grant permission and verify recording proceeds
- Deny permission and verify explanation screen shown
- Open settings button opens system settings
- Re-grant permission in settings, return to app, verify recording works
