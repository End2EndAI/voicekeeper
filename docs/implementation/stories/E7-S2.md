## Description

As a user, I need the note grid to filter in real-time as I type so that I can quickly find the note I am looking for.

## Acceptance Criteria

- [ ] Client-side filtering in NotesContext when `searchQuery` is non-empty
- [ ] Filter matches against `title`, `formatted_text`, and `raw_transcription`
- [ ] Matching is case-insensitive
- [ ] NoteGrid displays `filteredNotes` when search query is active, `notes` otherwise
- [ ] "No results found" message shown when search returns empty
- [ ] Filtering is fast (under 100ms for 100+ notes)
- [ ] Clearing search shows all notes again

## Technical Notes

### Filtering Logic in NotesContext
```typescript
const filteredNotes = useMemo(() => {
  if (!searchQuery.trim()) return notes;
  const q = searchQuery.toLowerCase();
  return notes.filter(note =>
    note.title.toLowerCase().includes(q) ||
    note.formatted_text.toLowerCase().includes(q) ||
    note.raw_transcription.toLowerCase().includes(q)
  );
}, [notes, searchQuery]);
```

### Empty State
```typescript
// In NoteGrid or Home screen
{filteredNotes.length === 0 && searchQuery ? (
  <View style={styles.emptyState}>
    <Text>No notes matching "{searchQuery}"</Text>
  </View>
) : null}
```
