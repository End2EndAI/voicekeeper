# Story: E2-S3 - Implement Logout

## Story Info
- **Epic:** E2 - Authentication
- **Sprint:** 1
- **Priority:** Must Have
- **Points:** 1
- **Status:** not_started
- **Dependencies:** E2-S2

## Description

As a user, I need to be able to log out so that I can switch accounts or secure my session.

## Acceptance Criteria

- [ ] Logout button accessible (temporarily in Home header until Settings screen is built)
- [ ] Tapping logout calls `supabase.auth.signOut()`
- [ ] AuthContext state clears (user and session set to null)
- [ ] App navigates to Login screen after logout
- [ ] Stored session tokens are cleared

## Technical Notes

### Implementation
```typescript
// In AuthContext
const signOut = async () => {
  await supabase.auth.signOut();
  // onAuthStateChange will handle state cleanup
};
```

### Temporary Placement
Add a simple logout icon/button in the Home screen header. This will be moved to the Settings screen in E6-S2.

## Files to Create/Modify
- `contexts/AuthContext.tsx` - Add signOut action (if not already)
- `app/index.tsx` - Add temporary logout button in header

## Testing
- Tap logout and verify redirect to login screen
- Verify session is cleared (close and reopen app, should see login)
- Verify no user data is accessible after logout
