# Story: E4-S4 - Build Note Preview Screen

## Story Info
- **Epic:** E4 - Transcription and Auto-Formatting
- **Sprint:** 2
- **Priority:** Must Have
- **Points:** 3
- **Status:** not_started
- **Dependencies:** E4-S3

## Description

As a user, I need to see my formatted note after processing and be able to edit it before saving, so that I can review and correct the output.

## Acceptance Criteria

- [ ] Preview screen (`app/preview.tsx`) displays after processing completes
- [ ] Loading overlay shown during processing with progress text:
  - "Transcribing your recording..." (while Whisper runs)
  - "Formatting your note..." (while GPT runs)
  - Or a single "Processing your note..." if stages cannot be distinguished
- [ ] Formatted text displayed with auto-generated title
- [ ] Format type badge shown (e.g., "Bullet List")
- [ ] Edit button enables text editing (TextInput in multiline mode)
- [ ] Title is editable
- [ ] Save button persists the note (delegates to note creation, wired in E5-S5)
- [ ] On error: show error message with Retry button
- [ ] On error fallback: display raw transcription if formatting failed
- [ ] Back/Cancel button returns to Home without saving

### Preview Screen State
```typescript
const [processing, setProcessing] = useState(true);
const [result, setResult] = useState<ProcessingResult | null>(null);
const [error, setError] = useState<string | null>(null);
const [editableTitle, setEditableTitle] = useState('');
const [editableText, setEditableText] = useState('');
const [isEditing, setIsEditing] = useState(false);

useEffect(() => {
  processAudio();
}, []);

const processAudio = async () => {
  try {
    setProcessing(true);
    setError(null);
    const result = await processRecording(audioUri, formatType);
    setResult(result);
    setEditableTitle(result.title);
    setEditableText(result.formatted_text);
  } catch (err) {
    setError(err.message);
  } finally {
    setProcessing(false);
  }
};
```
