# Story: E9-S3 - Accessibility and UI Polish

## Story Info
- **Epic:** E9 - Polish and Integration Testing
- **Sprint:** 4
- **Priority:** Must Have
- **Points:** 3
- **Status:** not_started
- **Dependencies:** None

## Description

As a user, I need the app to be accessible and visually polished so that it provides a good experience for all users including those using assistive technology.

## Acceptance Criteria

- [ ] All interactive elements have `accessibilityLabel` props
- [ ] All buttons have `accessibilityRole="button"`
- [ ] Text inputs have `accessibilityLabel` describing their purpose
- [ ] Color contrast meets WCAG AA (4.5:1 for normal text, 3:1 for large text)
- [ ] Screen reader navigation works on iOS (VoiceOver)
- [ ] Screen reader navigation works on Android (TalkBack)
- [ ] Card grid spacing is consistent (8px gaps)
- [ ] Typography is consistent (font sizes, weights, line heights)
- [ ] Loading states have accessible announcements
- [ ] Responsive layout works on:
  - Small phones (375px width)
  - Large phones (428px width)
  - Tablets (768px+ width)
- [ ] No visual glitches or overlapping elements
- [ ] Consistent color scheme throughout the app

## Accessibility Checklist

### Screens
- [ ] Login: form labels, error announcements
- [ ] Home: grid items navigable, record button labeled, search accessible
- [ ] Recording: stop/cancel labeled, timer announced
- [ ] Preview: loading state announced, edit/save labeled
- [ ] Note Detail: content navigable, edit/delete labeled
- [ ] Settings: format options selectable, current selection announced

### Components
- [ ] NoteCard: `accessibilityLabel` with title and preview
- [ ] RecordButton: `accessibilityLabel="Record new note"`
- [ ] FormatBadge: `accessibilityLabel` with format name
- [ ] SearchBar: `accessibilityLabel="Search notes"`
- [ ] AudioWaveform: `accessibilityLabel="Recording in progress"`
- [ ] LoadingOverlay: `accessibilityLabel` with current status

## UI Polish Checklist

- [ ] Consistent padding/margins (use 8px grid)
- [ ] Card shadows consistent across platforms
- [ ] Button press states (opacity or color change)
- [ ] Smooth transitions between screens
- [ ] Keyboard avoidance on forms (KeyboardAvoidingView)
- [ ] Safe area insets respected (notch, bottom bar)

## Files to Create/Modify
- All component files: add accessibility props
- All screen files: verify layout, spacing, accessibility
- Potentially create a shared `styles/` or `theme.ts` for consistent values

## Testing
- Enable VoiceOver on iOS, navigate through all screens
- Enable TalkBack on Android, navigate through all screens
- Use accessibility inspector to check contrast ratios
- Test on small and large screen sizes
- Visual review of all screens for polish issues
