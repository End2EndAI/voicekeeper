# Story: E2-S1 - Implement AuthContext and Session Management

## Story Info
- **Epic:** E2 - Authentication
- **Sprint:** 1
- **Priority:** Must Have
- **Points:** 3
- **Status:** not_started
- **Dependencies:** E1-S2

## Description

As a user, I need the app to manage my authentication state and persist my session so that I stay logged in across app restarts and am redirected to login when unauthenticated.

## Acceptance Criteria

- [ ] `AuthContext` created with state: `user` (User | null), `session` (Session | null), `loading` (boolean)
- [ ] `AuthProvider` wraps the app in `_layout.tsx`
- [ ] `supabase.auth.onAuthStateChange` listener updates context on login/logout/token refresh
- [ ] Session persists across app restarts (via SecureStore on mobile, localStorage on web)
- [ ] Auth guard in `_layout.tsx` redirects unauthenticated users to `/login`
- [ ] Authenticated users are redirected from `/login` to `/` (Home)
- [ ] Loading state shown while checking initial session

## Technical Notes

### AuthContext Implementation
```typescript
// contexts/AuthContext.tsx
interface AuthState {
  user: User | null;
  session: Session | null;
  loading: boolean;
}

interface AuthContextType extends AuthState {
  signUp: (email: string, password: string) => Promise<void>;
  signIn: (email: string, password: string) => Promise<void>;
  signOut: () => Promise<void>;
}
```

### Auth Guard Pattern
```typescript
// app/_layout.tsx
const { session, loading } = useAuth();

if (loading) return <LoadingScreen />;

return (
  <Stack>
    {session ? (
      <>
        <Stack.Screen name="index" />
        <Stack.Screen name="record" options={{ presentation: 'modal' }} />
        <Stack.Screen name="preview" />
        <Stack.Screen name="note/[id]" />
        <Stack.Screen name="settings" />
      </>
    ) : (
      <Stack.Screen name="login" />
    )}
  </Stack>
);
```

### Session Persistence
Already handled by the Supabase client config in E1-S2 (SecureStore adapter for mobile).
