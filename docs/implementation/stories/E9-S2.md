# Story: E9-S2 - Error Handling and Resilience

## Story Info
- **Epic:** E9 - Polish and Integration Testing
- **Sprint:** 4
- **Priority:** Must Have
- **Points:** 2
- **Status:** not_started
- **Dependencies:** None

## Description

As a user, I need the app to handle errors gracefully so that I never lose data and always know what went wrong.

## Acceptance Criteria

- [ ] Network failure during processing: show error message with Retry button
- [ ] Whisper API error: show error message, offer retry
- [ ] GPT formatting error: fall back to raw transcription as formatted_text
- [ ] Supabase save error: show error message, allow retry (data preserved in memory)
- [ ] Session expiry: redirect to login screen automatically
- [ ] Microphone permission revoked mid-session: handle gracefully
- [ ] No data loss on any failure path (audio preserved locally until processing succeeds)
- [ ] All error messages are user-friendly (no raw error codes)

## Error Scenarios to Test

| Scenario | Expected Behavior |
|---|---|
| Airplane mode during processing | "Network error. Check your connection and try again." + Retry |
| Server returns 500 | "Something went wrong. Please try again." + Retry |
| Whisper API rate limit | "Service is busy. Please wait and try again." + Retry |
| GPT returns malformed JSON | Fallback to raw transcription, note still saves |
| Supabase insert fails | "Failed to save note. Tap to retry." + Retry |
| Token expired mid-action | Redirect to login, show "Session expired" message |
| Storage upload fails | Note saved without audio (non-blocking) |

## Testing
